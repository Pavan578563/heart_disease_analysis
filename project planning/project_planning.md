# Heart Disease Analysis - Project Planning

## 1. Project Goals

*   To analyze large-scale heart disease data using Tableau (or Plotly + Flask).
*   To transform raw data into meaningful, interactive dashboards and stories.
*   To highlight key risk factors and identify correlations to support better decision-making for healthcare providers, policymakers, and individuals.
*   To uncover hidden trends and aid in preventive care and awareness.

## 2. Scope

The project will involve:

*   Data collection and extraction from a database.
*   Data preparation for visualization.
*   Creation of multiple unique data visualizations.
*   Development of responsive dashboards for three personas: cardiologist, government health department, and individual patient.
*   Creation of a data story with multiple scenes.
*   Performance testing of the dashboards.
*   Web integration of dashboards and stories using Flask.
*   Comprehensive project documentation.

## 3. Key Deliverables

*   Cleaned and prepared heart disease dataset.
*   SQL schema and operations for data storage.
*   Interactive Tableau dashboards (or Plotly + Flask web application).
*   Data story presenting key insights.
*   Project documentation (including problem understanding, development procedure, video script).
*   Web application for embedding dashboards.

## 4. Phases and Milestones

### Phase 1: Problem Understanding & Requirements (Completed)
*   **Milestone**: Define problem, specify business problem, business requirements, literature survey, social/business impact.

### Phase 2: Data Collection & Preparation
*   **Milestone**: Collect dataset, store data in DB, perform SQL operations, connect DB with Tableau, prepare data for visualization.

### Phase 3: Data Visualization & Dashboard Development
*   **Milestone**: Create unique visualizations, design responsive dashboards for different personas, create data story with multiple scenes.

### Phase 4: Performance Testing & Web Integration
*   **Milestone**: Conduct performance testing (data rendered, filter utilization, calculation fields, visualizations/graphs), embed dashboards and story with UI using Flask.

### Phase 5: Project Demonstration & Documentation
*   **Milestone**: Record explanation video, complete step-by-step project development procedure documentation.

## 5. Technology Stack

*   **Data Analysis**: Data Analysis, Visualization
*   **Business Intelligence**: Tableau (primary), Plotly (alternative for web integration)
*   **Database**: SQL
*   **Web Integration**: Flask, HTML, CSS, JavaScript
*   **Data Preparation**: Python (e.g., `data_preparation.py`)

## 8. Risks and Mitigation

*   **Risk**: Difficulty in accessing or cleaning large-scale health data.
    *   **Mitigation**: Allocate sufficient time for data engineering, utilize robust data cleaning scripts.
*   **Risk**: Tableau licensing issues or inability to share interactive dashboards widely.
    *   **Mitigation**: Develop a Plotly + Flask web application as an open-source alternative to ensure accessibility.
*   **Risk**: Dashboards not meeting user-specific needs.
    *   **Mitigation**: Regular feedback sessions with target personas (cardiologist, policymaker, individual) during development.
